# Streamlit Main Site 2.0 升级方案：打造全能思维驾驶舱

我们目前的 Main Site 像一个“记账本”（只记录数量），我们要把它升级为您的“**首席思维顾问**”（提供洞察与建议）。

针对您提到的“功能不全”和“必须支持英文”，我提出以下 **3 大维度** 的深度升级建议：

## 1. 核心引擎升级：双语 NLP 与智能分析 (Intelligence)
这是最底层的升级，解决“看不懂英文”和“分析太浅”的问题。

*   **双语分词系统 (Bilingual NLP)**:
    *   **现状**: 只有 `jieba`（中文），英文分词一团糟。
    *   **升级**: 引入 `nltk` 库。
        *   **英文去噪**: 自动过滤 "the", "is", "a", "please" 等无意义停用词。
        *   **词形还原**: 把 "coding", "codes" 统一识别为 "code"。
*   **思维复杂度评分 (Complexity Score)**:
    *   **新功能**: 不只看你写了多少，更看你写得**有多深**。
    *   **算法**: 基于 Prompt 长度、逻辑连接词（因为、所以、if、however）、专业术语密度，给每条 Prompt 打分 (0-100)。
    *   **价值**: 帮您识别出那些“高质量思考”的时刻。

## 2. 可视化叙事升级：从“看图表”到“看故事” (Storytelling)
图表不应该只是数据的堆砌，而应该讲述您的大脑如何运作。

*   **GitHub 风格思维热力图 (Habit Heatmap)**:
    *   **建议**: 放弃普通的柱状图，使用 **Calendar Heatmap**（类似 GitHub 的绿格子）。
    *   **价值**: 一眼看清您哪天思维最活跃，哪个月在“摸鱼”，形成连续的打卡激励。
*   **动态技能雷达 2.0**:
    *   **升级**: 建立中英双语关键词映射表（例如 `Code` = `代码` = `编程`）。
    *   **价值**: 无论您用中文还是英文提问，都能精准归类到“编程”、“写作”或“逻辑”维度。

## 3. UI/UX 体验重构：产品级质感 (Experience)
*   **深色模式适配 (Dark Mode)**: 自动检测系统设置，支持手动切换，配色对齐插件的蓝紫色调。
*   **交互式过滤器**: 增加侧边栏过滤器，允许您按“复杂度”、“分类”或“时间段”筛选查看 Prompt。

## 🛠️ 技术执行计划 (Implementation Plan)

### Phase 1: 基础建设 (Infrastructure)
1.  **依赖更新**: 修改 `requirements.txt`，添加 `nltk` (自然语言处理库)。
2.  **数据清洗**: 重写数据加载逻辑，增加英文停用词过滤列表。

### Phase 2: 核心功能开发 (Features)
3.  **双语适配**: 更新 `get_words` 函数，同时支持 `jieba` (中) 和 `nltk` (英)。
4.  **雷达图重构**: 更新关键词词典，支持中英双语映射。
5.  **热力图实现**: 引入 `streamlit-calendar` 或使用 `plotly` 绘制日历热力图。

### Phase 3: 界面美化 (UI Polish)
6.  **样式注入**: 编写 Custom CSS，美化 Metric 卡片和 DataFrame 表格。
7.  **布局优化**: 采用 `st.columns` 和 `st.expander` 优化信息密度。
