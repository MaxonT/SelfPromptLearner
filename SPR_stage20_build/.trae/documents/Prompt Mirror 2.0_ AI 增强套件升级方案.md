# Prompt Mirror 1.5: 稳健增强 (Robust Enhancement)

**核心原则**：
**保留所有现有优点**（不删滚动条，不改动核心布局），在现有基础上做**微创手术**，提升质感和准确度。

## 1. 交互微调：丝滑展开 (Silky Expansion)
*   **现状**：点击列表项时，高度变化比较生硬。
*   **优化**：
    *   保留原有 DOM 结构。
    *   在 CSS 中为 `.prompt-text` 的 `max-height` 属性添加 `transition` 过渡。
    *   这就够了。不引入复杂的 JS 物理引擎，用最原生的 CSS 实现最稳健的动效。

## 2. 数据增强：分类精度提升 (Classification Refining)
*   **现状**：目前的 `CATEGORIES` 字典比较基础，容易把一些深层问题归类为「其他」。
*   **优化**：
    *   **扩充关键词库**：基于常见 Prompt 模式，增加「架构」、「调试」、「优化」、「翻译」等细分词根。
    *   **优先级逻辑**：比如同时出现 "代码" 和 "解释"，优先归类为 "学习" 而不是 "编程"（因为意图是理解而非执行）。
    *   **不删减**：保留原有分类，只是让匹配更精准。

## 3. 视觉微调：字体与排版 (Typography)
*   **数字对齐**：给 KPI 卡片里的数字设置 `font-feature-settings: "tnum"` (Tabular Numbers)，确保数字变化时宽度不变，视觉更稳定。
*   **空状态优化**：给 "还没有数据" 的提示加一个微小的引导箭头，指向右上角的扫描按钮，让新用户一眼知道该点哪里。

## 执行清单
1.  **CSS**: 增加 `transition` 和 `font-feature-settings`。
2.  **JS**: 扩充 `CATEGORIES` 对象，微调 `classify` 函数的判断顺序。
3.  **HTML**: 在空状态 (`empty-state`) 里加一个小箭头 SVG。

**这个方案只做「加分项」，不做「减法」。**
